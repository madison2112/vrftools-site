Central Controller Config Tools

Web-based utilities for Mitsubishi Electric AE-200 and AE-C400A controller configuration files.

About These Tools

The central controller config file required to load group and address data into an AE-200 or AE-C400A already exists in your DSB file — it just isn't in the right format. The DSBX to DAT converter closes that gap, turning a .dsbx file into one loading-ready config file per centralized system with no manual data entry. The remaining tools handle the next most common pain points: reordering groups after the fact, converting between controller generations, and splitting a site database into individual controller files that can actually be loaded when the IT backbone isn't complete yet.

All processing happens on the server. Uploaded files are automatically deleted after one hour. Maximum file size: 5 MB. See the How to Use guide for detailed instructions.

DSBX to DAT
.dsbx → one or more .dat

The problem: Building a central controller config file requires manually entering the name and M-Net address(es) for every group. This takes a significant amount of time and is a source of frustration — the data already exists in the DSB file.

The solution: Upload your .dsbx file and download fully built config file(s) ready to be opened with the Initial Settings Tool. Groups are auto-populated from the DSB data, and you can optionally reorder them before downloading. Generates one config file per centralized system.
⇅
Group Rearranger
.dat → .dat

The problem: Units display on the controller touchscreen in group order. When groups don't match the preferred display order — as is often the case when there are multiple systems on one floor — fixing the group ordering requires significant editing to the config file or rebuilding the database entirely.

The solution: Upload your config file and drag groups into the order you want. Download a modified config file ready to be loaded to the controller.
⇆
Generation Change
AE-200 ⇄ AE-C400A

The problem: The AE-200 and AE-C400A (and their EW counterparts) use different .dat formats that aren't cross-compatible. If a database was built for the wrong controller generation, or a central controller is being upgraded or replaced, the only option is to rebuild the entire database.

The solution: Upload a config file and download it converted to the opposite generation. Group data, names, and M-Net assignments are preserved automatically.
⌫
Database Decoupler
multi .dat → multiple .dat

The problem: A multi-controller config file can only be loaded when all controllers in the database are already communicating with each other. On sites where the IT backbone isn't yet complete, individual controllers can't have their databases loaded without separate config files for each controller.

The solution: Upload a config file with multiple central controllers and download a ZIP of individual per-controller files, each ready to load independently.
